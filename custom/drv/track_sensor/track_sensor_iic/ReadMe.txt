
	G-sensor IIC通信说明

----------------------------------------------------------------------------------------------------------

1、BMA250/E的驱动完全为厂家提供，其代码被封闭，强制修改IIC通信代码存在风险，单独处理，不作变动

2、MC3410/AFA750/MMA8452Q只是从厂家获取到IIC通信，其他配置完全自己实现，可以作合并优化

3、芯片手册和实际测试表明，MC3410/AFA750/MMA8452Q 三个G-sensor的通信，都是标准的IIC通信，其通信代码可以单独列出共用

4、IIC通信的地址，每个Sensor各不相同，而且可能与硬件脚位相关，请注意区分

5、IIC通信的时钟，在不同MTK平台系统时钟不同，因对IIC时钟延时作调整

6、理论上未来增加其它Sensor 可以摆脱无厂家代码的限制，完全自定义实现功能，除非新sensor的通信比较另类
